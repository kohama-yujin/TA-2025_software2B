libMessage.aを削除し、プログラムを実行したところ、問題なく動作した。
しかし
libMessage.soを削除し、プログラムを実行したところ、
./report3_dynamic: error while loading shared libraries: libMessage3.so: cannot open shared object file: No such file or directory
とだけ出力され、動作しなかった。

これは静的リンクライブラリと動的リンクライブラリの違いから起こる現象で、
静的リンクライブラリは、プログラムを生成する際、直接プログラムにリンクするため、後にライブラリが削除されても動作する。
一方、動的リンクライブラリは、プログラムを実行する際にリンクするため、実行時にライブラリが存在しないとエラーとなる。

静的リンクライブラリ
利点：
　・一度リンクしてしまえばプログラムのみで実行可能
欠点：
　・ファイルサイズが大きくなる
　・ライブラリが更新された場合にプログラムをコンパイルし直さないといけない

動的リンクライブラリ
利点：
　・ファイルサイズが小さくなる
　・ライブラリが更新されても、コンパイルし直すことなく実行することができる
欠点：
　・関数の呼び出しに時間がかかる
